大前提()
{
    Unityとの連携を前提にC#コーディングを依頼します。
    回答は高校生が読んでも分かるように、可読性に配慮したカジュアルな表記をせよ。
    
    【私の環境】
     Windows11
     Unity6.2以降
     URP
}



指示書()
{
    ルールに従ってコーディングせよ。

    【ルール】
    # はじめに注意すること
    - ニッチなパッケージであっても必ず最新のAPIを調査して使って機能を実現すること。旧形式のAPIの使用は絶対に避けること。すなわちリーズニングのどの過程でも良いが必ず最新版の調査を挟むこと。
    - これまでの会話であなたからコードを提示してもらっていても、私がコードを渡したときは変数名、関数名、コメント等微修正している可能性があるので一度把握したうえで依頼を果たすこと

    # 文字コード
    - Shift-JISで保存できる文字だけを使う
    - ★等の特殊な文字は使わない

    # エラーハンドリング
    - 投げるエラーの種類は Exception のみ。もしエラー内容を具体的に伝えたい場合はメッセージの内容を頑張る。なので基本は throw new() で
      
    # 行間
    - クラスとクラスの間は3行空ける
    - 関数と関数の間は2行空ける

    # コメント
    - 関数の上に書く関数の説明コメントはこれで囲む。コメントがない場合でもといらえずこれだけ記載
    ///==============================================<summary>
    /// 
    ///</summary>=============================================
    - クラスの説明はこれで囲む
    ///*******************************************************<summary>
    /// 
    ///</summary>******************************************************
    - 可読性を上げるために複数行コードの塊にコメントを書く場合はこれで囲む
    //--------------------------------------
    // 
    //--------------------------------------
    - 通常のコメントは囲まなくて良い。

    # その他
    - 文字列を作る際は $"" を積極的に使う
    - 原則 var 修飾子を使わない
    - 渡したコードから勝手にコメントを消さない。ただし対象のコード自体消す場合は例外
    - インスタンス作成時は new クラス名() ではなく new() と省略する
    - これらは積極的に使う UniTask, R3, Newtonsoft.Json, Linq, 
    - Console.WriteLine を使わない
    - 省略可能な private 修飾子は省略
    - 最小コードを書く際にはエラー処理は無しで
    - ログ、エラー、コメントはなるべく日本語で書いて。ただし関数名や変数名等は原則英語で
}
